Sử dụng GitHub Actions là một lựa chọn cực kỳ thông minh vì nó miễn phí (cho repo public hoặc có giới hạn giờ cho repo private) và tích hợp sẵn với luồng code của bạn.
Dưới đây là kế hoạch chi tiết để "Automate" việc train model của bạn:
1. Kiến trúc hệ thống
1.
Trigger: Bạn thiết lập một Supabase Webhook. Khi có dòng dữ liệu được chuyển sang approved, Webhook này sẽ gọi đến GitHub API (Repository Dispatch) để kích hoạt workflow.
2.
GitHub Worker:
◦
Cài đặt môi trường Python & TensorFlow.
◦
Chạy script tải toàn bộ ảnh approved từ Supabase Storage về máy ảo.
◦
Thực hiện Transfer Learning (Huấn luyện lại dựa trên model cũ) để tiết kiệm thời gian.
◦
Xuất file model_unquant.tflite mới.
3.
Deploy: Tự động đẩy file model mới lên Supabase Storage (vào một folder tên là models/latest/).
2. Cấu hình GitHub Workflow

Bước cuối cùng bạn cần làm để kích hoạt hệ thống:
1.
Cấu hình GitHub Secrets: Vào Repo của bạn trên GitHub > Settings > Secrets and variables > Actions. Thêm các biến sau lấy từ file .env của bạn:
◦
SUPABASE_URL
◦
SUPABASE_ANON_KEY
◦
SUPABASE_SERVICE_ROLE_KEY (Cái này lấy trong Supabase Dashboard > Settings > API).
2.
Tạo Webhook trên Supabase: Vào Supabase Dashboard > Database > Webhooks. Tạo một Webhook:
◦
Table: waste_submissions
◦
Events: UPDATE
◦
Condition: status bằng approved.
◦
Target: Gửi một POST request đến GitHub API: https://api.github.com/repos/YOUR_USERNAME/YOUR_REPO/dispatches
◦
Headers:
▪
Authorization: Bearer YOUR_GITHUB_TOKEN
▪
Accept: application/vnd.github.v3+json
◦
Body: {"event_type": "submission_approved"}.
